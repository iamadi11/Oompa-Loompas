# ADR: Progressive Web App for the web client

**Date:** 2026-04-06  
**Status:** Accepted  
**Authority:** SOURCE_OF_TRUTH.md v1.1.0 — §7.8

## Context

The product web client (`apps/web`) should be installable on supported platforms, offer a reliable app shell, and follow the UI principles in SOURCE_OF_TRUTH §7 (intentional, human-crafted, minimal motion, WCAG 2.2 AA). Financial data must remain deterministic: no silent stale deals, payments, or balances when offline.

## Decision

1. **Ship the Next.js web app as a PWA** using a Web App Manifest, maskable icons, and a **service worker** scoped to static precaching and offline fallback — not to caching API responses for revenue paths.

2. **Tooling:** **`@serwist/turbopack`** with **`withSerwist`** in `next.config.mjs`, worker source **`app/sw.ts`**, and **`/serwist/sw.js`** served via **`app/serwist/[path]/route.ts`**. Service worker **registration disabled in development** (`SerwistRegister` / `SerwistProvider`) to preserve fast iteration and predictable dev refresh. Uses **default Next.js 16 bundling (Turbopack)** for `next dev` and `next build` — no webpack-only PWA plugin.

   **Amendment (2026-04-07):** Supersedes the initial `@ducanh2912/next-pwa` choice; caching and API policy below are unchanged.

3. **Caching policy:**
   - **Precache:** Next static assets and the offline fallback page as generated by the integration.
   - **API / dynamic data:** **Network-only** (or equivalent): do not use stale-while-revalidate or cache-first for `/api/*` or other endpoints used for deals, payments, or money-adjacent state.

4. **Offline UX:** When navigation or data fetch fails due to connectivity, show explicit copy that **revenue actions require a connection**; do not imply current screen data is valid money truth.

## Consequences

**Positive:** Installability, faster repeat visits for static shell, clearer product positioning as a “real” client.

**Negative:** Service worker adds build complexity; must be verified on production builds; team must avoid expanding cache rules without ADR update.
